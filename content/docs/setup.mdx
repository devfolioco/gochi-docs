Four steps: run the installer, clone the repo, pull down the ESP32
core, then flash a tiny LED-blink test to confirm the toolchain works.

<Callout type="info">
  This page gets your toolchain ready. The
  [Wiring](/docs/wiring) page is where you build the pet itself and flash
  the real firmware.
</Callout>

<Steps>

<Step>

## Install the toolchain

The installer pulls everything you need: `make`, `git`, `arduino-cli`,
and `bun` (plus Homebrew on macOS or Chocolatey on Windows if they're
missing). It's idempotent — anything you already have is left alone.

<Tabs items={['macOS / Linux', 'Windows']}>

<Tab value="macOS / Linux">

```sh
curl -fsSL https://gochi.in/install.sh | bash
```

</Tab>

<Tab value="Windows">

Open **PowerShell as Administrator** (Chocolatey needs it), then run:

```ps
powershell -c "irm gochi.in/install.ps1 | iex"
```

</Tab>

</Tabs>

<Callout type="info">
  Want to read the script first? It's the right thing to do.
  [`install.sh`](/install.sh) · [`install.ps1`](/install.ps1).
</Callout>

</Step>

<Step>

## Clone the repo

**Recommended:** fork first. Head to
[github.com/devfolioco/gochi](https://github.com/devfolioco/gochi),
click **Fork**, and clone your fork (replace `YOUR-USERNAME`):

```sh
git clone https://github.com/YOUR-USERNAME/gochi.git
cd gochi
```

Forking means you can push your tweaks (custom faces, new sounds, IMU
gestures) without needing write access to the main repo, and PRs are
one click away.

Just want to try it? Clone upstream directly:

```sh
git clone https://github.com/devfolioco/gochi.git
cd gochi
```

Every command from here on runs from the repo root.

</Step>

<Step>

## Install the ESP32 core

A ~300 MB download — grab a coffee. The repo ships its own
`arduino-cli.yaml`, so this command is the same on every machine:

```sh
arduino-cli --config-file firmware/arduino-cli.yaml core update-index
arduino-cli --config-file firmware/arduino-cli.yaml core install esp32:esp32
```

</Step>

<Step>

## Plug the ESP32-C3 in, verify flashing works

A small **red power LED** lights up. Use a **USB-C data cable**, not a
power-only one.

```sh
make test-led
```

The port is auto-detected. If the upload hangs at *"Connecting…"*, hold
**BOOT**, tap **RESET**, release **BOOT**, and run it again. Once it
completes, the **blue LED** on the ESP32-C3 starts blinking — flashing
works.

</Step>

</Steps>
