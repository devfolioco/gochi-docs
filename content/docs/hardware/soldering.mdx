The kit ships as two unpopulated boards. By the end of this page you'll
have one assembly: the **SSD1306 OLED** sitting on a 4-pin header that
plugs into the **ESP32-C3 SuperMini** via four jumpers.

## What you need

| Item                  | Notes                                                |
| --------------------- | ---------------------------------------------------- |
| Soldering iron        | Anything 30–60W with a fine tip. We provide these.   |
| Solder                | 0.6–0.8mm leaded rosin-core is easiest for beginners |
| Flux pen (optional)   | Makes joints cleaner; not required                   |
| Wire cutters          | For trimming pin leads after soldering               |
| Helping hands / vise  | Optional but very nice for the header step           |

## Pinout

Memorize this table. Everything else is just connecting it up.

| OLED pin | ESP32-C3 pin | Signal           | Wire color (convention) |
| -------- | ------------ | ---------------- | ----------------------- |
| `GND`    | `G`          | Ground           | Black                   |
| `VCC`    | `3V3`        | 3.3V power       | Red                     |
| `SCL`    | `GP5`        | I²C clock        | Yellow                  |
| `SDA`    | `GP4`        | I²C data         | Blue                    |

<Callout type="warn">
  **VCC goes to 3V3, not 5V.** The SSD1306 boards in the kit run on 3.3V.
  5V will damage them. The ESP32-C3 SuperMini has both pins labeled — use
  the one that says **3V3**.
</Callout>

## Soldering the OLED header

<Steps>

<Step>

**Insert the header into the OLED from the bottom side.** The four short
pins poke up through the holes, the longer pins (and the black plastic)
stay underneath. The screen faces up, header pins poke out the back.

If your OLED came with the header pre-soldered, skip ahead to
[the wires](#running-the-wires).

</Step>

<Step>

**Lay the OLED face-down on a flat surface** so the header presses flush
against the board. Helping hands work too — anything to keep the parts
from wobbling.

</Step>

<Step>

**Heat one pin from the top side** (the side with the labels, opposite
the screen). Touch the iron to the pin and the pad at the same time for
about a second, then feed solder into the joint until it forms a smooth
cone. Remove solder first, then the iron.

A good joint looks like a shiny volcano. A bad one looks like a dull ball
or a spike — reflow with more flux/heat if so.

</Step>

<Step>

**Do the other three.** Don't rush. Wait a few seconds between pins to
let things cool.

</Step>

</Steps>

## Running the wires

You can either solder jumpers directly between the boards, or plug the
OLED into a breadboard and run breadboard jumpers across. The workshop
default is **direct solder** — fewer parts, more durable.

<Steps>

<Step>

**Cut four jumper wires** to about 80mm. Strip 4mm at each end and tin
the tips (touch them to a small bead of solder so the strands hold
together).

</Step>

<Step>

**Solder the OLED end first.** Match colors to the pinout table:

- Black → `GND` pad
- Red → `VCC` pad
- Yellow → `SCL` pad
- Blue → `SDA` pad

Solder from the underside of the OLED board — the same side as the header
plastic. Keep the joints small so the board can still sit flat.

</Step>

<Step>

**Now the ESP32 end.** Route each wire to its matching pin on the
SuperMini and solder through the hole. Trim excess pin length after.

- Black → `G`
- Red → `3V3`
- Yellow → `GP5`
- Blue → `GP4`

</Step>

<Step>

**Visual check.** Hold the assembly up to a light. Every joint should be
shiny, the wire fully embedded in the cone of solder, no stray strands
shorting to neighboring pads.

</Step>

</Steps>

## Sanity check before flashing

Before you plug it into a laptop:

- No two adjacent pins are bridged by solder. Slide a pin or a fingernail
  between them.
- The OLED's `VCC` line goes to **`3V3`** on the ESP32, not `5V`.
- The ESP32-C3 isn't reversed in its USB-C port (the chip side faces up
  on the SuperMini, with the antenna trace on the right).

If everything checks out, head to [Flashing the firmware](/docs/firmware/flashing).

<Callout type="info">
  **Pro tip:** before you flash anything, plug the ESP32 in by itself and
  watch the onboard blue LED on GPIO8. If it doesn't pulse on first
  power, you have a USB/power problem, not a code problem.
</Callout>

## Troubleshooting

| Symptom                                  | Likely cause                                              |
| ---------------------------------------- | --------------------------------------------------------- |
| Screen stays black after flashing        | `SDA`/`SCL` swapped, or wires loose                       |
| Screen shows scrambled noise             | I²C address mismatch — check the [firmware page](/docs/firmware/flashing#oled-i²c-address) |
| ESP32 doesn't enumerate as a USB device  | Cable is power-only; try another USB-C cable              |
| Smoke or burnt smell                     | Stop, unplug. Most likely VCC→5V instead of 3V3           |
